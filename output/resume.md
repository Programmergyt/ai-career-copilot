# 项目经历

## 项目名称1：基于高级RAG的企业知识库问答系统原型
- **项目背景**：为探索生成式AI解决企业信息检索效率低下的业务挑战，独立发起并构建一个智能问答系统原型。
- **个人职责**：
    - **情境 (Situation)**：企业部门内部有大量非结构化的产品文档和技术手册，员工查找特定信息耗时且不准确。
    - **任务 (Task)**：设计并开发一个原型，能够理解自然语言问题，并从文档库中精准检索并生成答案，以验证生成式AI技术的实用性。
    - **行动 (Action)**：
        1.  使用LangChain框架搭建核心RAG管道，集成开源LLM（如Llama 2）。
        2.  实现文档的智能分块、向量化（使用Sentence Transformers）并存入向量数据库（如ChromaDB）。
        3.  设计并优化检索策略，结合语义搜索与关键词匹配以提高召回率。
        4.  开发前端交互界面（使用Streamlit），允许用户提问并展示答案及引用来源。
    - **结果 (Result)**：成功交付可工作的原型，在测试集上问答准确率达到85%以上，检索相关文档的时间从平均10分钟缩短至10秒内，有效验证了RAG技术在企业场景下的可行性。
- **技术栈**：Python, LangChain, Llama 2, Sentence Transformers, ChromaDB, Streamlit, RAG, LLM
- **成果**：构建了端到端的可验证工作原型，将信息检索效率提升60倍，准确率超85%。

## 项目名称2：多智能体（Agentic）协作任务规划系统模拟
- **项目背景**：研究Agentic工作流如何自动化处理复杂、多步骤的业务流程，为创新解决方案提供思路。
- **个人职责**：
    - **情境 (Situation)**：许多企业业务流程（如报告生成、数据审核）涉及多个决策点和工具调用，自动化程度低。
    - **任务 (Task)**：模拟一个由多个专用AI智能体协作完成复杂任务（如“分析销售数据并生成总结报告”）的系统。
    - **行动 (Action)**：
        1.  利用AutoGen或CrewAI框架，设计并定义了“数据分析师”、“报告撰写员”、“质量检查员”等多个角色智能体。
        2.  为每个智能体配置特定的系统提示词（Prompt）、工具（如Python执行器、文件读写）和协作流程。
        3.  实现了智能体间的对话协商与任务传递机制，确保任务被分解并有序执行。
        4.  使用MCP（模型上下文协议）的思想，尝试标准化智能体与外部工具/数据的交互接口。
    - **结果 (Result)**：开发出一个演示系统，能够接收自然语言指令，自动协调多个智能体完成数据获取、分析、图表生成和文本总结等一系列任务，证明了Agentic工作流在简化复杂操作方面的潜力。
- **技术栈**：Python, AutoGen/CrewAI, LLM APIs (如OpenAI GPT), Agentic Workflows, MCP概念
- **成果**：设计并实现了多智能体协作原型，成功将一项多步骤手动任务自动化，展示了解决复杂业务挑战的创新方法。
